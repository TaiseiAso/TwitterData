Twitter API Key は各自取得して、
config/twitter.yml に、以下の形式で保存してください。

twitter_API:
  consumer_key        : xxxxx
  consumer_secret     : xxxxx
  access_token        : xxxxx
  access_token_secret : xxxxx

(xxxxxの部分に取得したkeyの値を入れる。)

-------------------------------------------------------------

$ python twitter.py
を実行すると、
data/ 下に、対話データが蓄積されていきます。
Ctrl+C で安全に終了できます。

$ python clear.py
を実行すると、
収集したデータや一時保存ファイルをすべて削除します。

-------------------------------------------------------------

さまざまな正規化処理を施しているため、比較的きれいな文章が保存されます。
 - 絵文字や顔文字の除去
 - あらゆる記号の正規化
 - ツイッター用語などの特定の語を含む対話の除去
 - ハッシュタグやURL、画像などの外部情報を含む対話の除去
 - 、。！？...などの出現をルール化
 - 文末表現は、「。」「！」「？」「!?」の４種類
など。

-------------------------------------------------------------

data/twitter.txt:
	二話者による複数行の複数ターン対話。

data/twitter_input.txt, data/twitter_target.txt:
	twitter.txt のすべての発話ペアを切り取って、入力と目標に分けて保存したもの。
	1 -> 2 -> 3 -> 4 という複数ターン対話があれば、
	1 -> 2, 2 -> 3, 3 -> 4 という複数の発話ペアに分けられる。

data/twitter_many_input.txt, data/twitter__many_target.txt:
    twitter_input.txt と twitter_target.txt を一文ごとに改行で区切ったもの。
    データ間はさらに改行で分割されている。

-------------------------------------------------------------

収集される対話データの文の長さはバラバラなので、必要に応じて長い文を除去したりしてください。

単純計算で大体1週間で約100万の発話ペア(約50万の複数ターン対話)が収集できるはず。
